一、考核出题思路
本次《前端开发技术》期末考核旨在全面检验学生对 Vue 前端技术栈的综合应用能力，
重点评估组件化开发、状态管理、路由配置及第三方 API 集成（如 AI 大模型）等核心实
践技能。考核设计严格遵循"理论+实践"的双轨导向，既注重对前端开发基础理论的理解深
度，也强调技术落地的实际操作能力。
出题思路核心原则：结合当前前端开发技术趋势，将传统功能模块（包括但不限于登录
注册系统、后台数据管理界面）与 AI 技术应用场景进行有机融合，通过真实业务场景驱动
技术能力考核，体现前沿性与实用性的平衡统一。
考核内容设计为后续作品开发范围与功能要求奠定逻辑基础，要求学生在规定时间内
完成从需求分析、架构设计到代码实现的完整开发流程，全面展示前端工程化思维与问题解
决能力。
二、作品设计范围 
本次项目要求开发一个高度模块化、可扩展的 Web 系统平台，该系统不局限于特定内容
主题，可以完成类似于论坛系统、游戏门户、社交网站、影视资源站、小说阅读平台或动漫
社区等多种类型的作品。平台必须完整实现用户注册与登录、个人中心信息管理、会员等级
与权益体系等核心账户功能，并深度集成 AI 能力，例如在用户生成内容（UGC）中提供 AI
辅助创作（如 AI 生成帖子、评论或简介），以及部署一个能够智能解答用户关于平台使用
和规则问题的 AI 客服机器人。作品重点考察需求挖掘的能力，产品设计的能力以及解决用
户痛点的能力，能够实现逻辑闭环的能力。
三、作品设计要求 
1.首页模块
基础要求：页面结构需拆分为 Header（导航栏）、Banner（轮播图）、Feature（功能
入口区）、Footer（页脚）四个独立 Vue 组件，确保响应式布局适配移动端与桌面端。交
互逻辑上实现导航栏滚动时的样式变化（透明度/背景色切换），Banner 轮播图支持自动播
放与手动切换功能。技术要点包括使用 CSS Flexbox 实现组件布局，通过 Vue Transition 
实现轮播图切换动画。
进阶要求：采用组件懒加载 策略优化首屏加载速度，将非关键组件（如 Footer）设置
为异步加载。通过 Vuex 管理全局主题状态，支持用户切换深色/浅色模式并持久化保存偏
好设置。代码层面需实现 组件复用，将 Button、Card 等基础元素封装为全局组件，提升
代码可维护性。

2.登录注册模块
基础要求：页面结构拆分为 LoginForm 与 RegisterForm 两个表单组件，共用 Captcha
（验证码）子组件。交互逻辑需使用 VeeValidate 实现表单验证，包含用户名格式（4-16 
位字母数字组合）、密码强度（至少 8 位含大小写字母与特殊符号）、手机号格式等校验
规则，验证失败时显示具体错误提示。技术要点包括通过 Axios 发送登录请求，使用
localStorage 存储认证令牌（Token）。
进阶要求：实现 表单状态管理，通过 Vuex 同步登录状态至全局，避免页面刷新后状
态丢失。添加 登录安全机制，包括密码输入框显示/隐藏切换、连续登录失败 5 次后触发
图形验证码、登录成功后的路由守卫验证。代码层面采用 Composition API 重构表单逻辑，
提取复用性高的验证规则与 API 调用函数。
3.个人中心模块
基础要求：页面结构分为 UserProfile（基本信息展示）、ProfileEditor（信息编辑
表单）、OrderList（订单记录）三个主要组件。交互逻辑支持用户修改昵称、头像、联系
方式等信息，修改后通过 Vuex 同步更新全局用户状态，并使用 防抖函数 优化输入框实时
验证性能。技术要点包括使用 Vue Router 实现个人中心子路由跳转，通过 Promise 处理
多接口并发请求（如同时获取用户信息与订单数据）。
进阶要求：实现 数据缓存策略，使用 localStorage 缓存用户基本信息，减少重复请
求。添加 信息修改权限控制，敏感操作（如修改手机号）需二次验证密码或短信验证码。
UI 层面优化表单交互体验，如头像上传支持拖拽排序、生日选择器使用日期组件、地址管
理采用三级联动选择器。
4.会员中心模块
基础要求：页面结构包含 MembershipCard（会员信息卡片）、PrivilegeList（权益列
表）、RenewalForm（续费表单）组件。交互逻辑实现会员等级展示（普通/白银/黄金）、
剩余时长倒计时、权益图标点击查看详情功能。技术要点包括使用 Vuex 管理会员状态，通
过 Axios 调用会员权益接口，使用 v-if/v-else 控制不同等级会员的权益展示。
进阶要求：设计 会员成长体系，通过折线图展示成长值变化趋势（使用 ECharts 实
现），支持查看历史消费记录与积分明细。添加 会员推荐功能，生成专属邀请链接并统计
邀请人数。性能优化方面采用 虚拟滚动 加载长列表权益内容，避免一次性渲染大量 DOM 
节点。
5.AI 功能模块
基础要求：页面结构需包含 ChatInterface（对话界面）、MessageList（消息列表）、
InputArea（输入区域）、HistoryPanel（历史记录面板）四个核心组件。交互逻辑支持用
户输入文本/上传文件（限 TXT/PDF 格式）、发送/取消发送、清空对话等基础操作，使用
v-for 动态渲染消息列表，通过 v-bind:class 区分用户/AI 消息样式。
技术要点：
自然语言交互界面：输入框支持 Enter 发送消息、Shift+Enter 换行，历史记录面板
实现会话切换与删除功能；API 接口封装：使用 Axios 创建专用请求实例，配置请求拦截
器添加认证 Token，响应拦截器统一处理 401（未授权）、503（服务不可用）等错误状态
码；响应结果渲染：对 AI 返回的 Markdown 格式内容，使用 marked 库转换为 HTML，通
过 v-html 指令渲染富文本，同时使用 DOMPurify 过滤恶意代码防止 XSS 攻击。
进阶要求：实现 多轮对话上下文理解，通过 Vuex 存储上下文 ID 并在请求时携带，
支持上下文回溯与修改。添加 对话功能扩展，包括消息收藏/标签分类、AI 回答内容的复
制/导出（支持 Markdown/PDF 格式）、自定义对话角色（如“代码助手”“写作导师”）。
技术层面采用 WebSocket 实现实时消息推送，替代传统轮询机制；使用 Pinia 替代 Vuex 
实现更细粒度的状态管理；通过 Web Worker 处理大型文本解析，避免阻塞主线程。
6.后台管理系统
基础要求：页面结构采用 Layout 布局组件，包含 Sidebar（侧边导航）、Header（顶
部操作栏）、MainContent（主内容区）三个部分。交互逻辑实现侧边栏折叠/展开、面包屑
导航、用户退出登录功能。技术要点包括使用 Vue Router 实现路由嵌套（如 /admin/user、
/admin/order），通过 路由元信息（meta）控制页面权限，未登录用户自动重定向至登录
页。
进阶要求：设计 权限管理系统，基于用户角色（超级管理员/普通管理员/运营人员）
动态生成侧边栏菜单，通过 路由守卫 实现页面级权限控制。添加 数据可视化仪表盘，使
用 ECharts 展示用户增长、订单统计等核心指标，支持按日/周/月切换数据维度。性能优
化方面实现 路由懒加载 与 组件缓存（keep-alive），减少重复渲染开销；通过 Tree Shaking 减小打包体积，提升加载速度。
各模块设计需遵循 单一职责原则，基础要求确保核心功能可用，进阶要求通过技术优
化提升用户体验与代码质量，整体架构需支持未来功能扩展与性能升级。

五、评分细则 
1.项目基础与主题 (10 分)：项目至少包含 3 个由 Vue Router 管理的路由页面，页面
内容与所选主题高度相关、健康向上，能通过导航无缝切换，形成完整的单页面应用(SPA)
体验。
2.组件化页面结构 (10 分)：每个页面（页面级组件）均采用组件化思想构建，结构清
晰，合理运用<header>、<nav>、<main>、<footer>等语义化标签或相应的布局组件，清晰
划分头部、导航、焦点内容区、底部等区域。
3.Vue 单文件组件(SFC)与样式分离 (5 分)：使用 Vue 单文件组件（.vue 文件）开发，
在<style>标签内编写 CSS 代码，或通过@import 引入外部 CSS 文件，完美实现结构
（<template>）、逻辑（<script>）与样式（<style>）的分离。
4. 创意性与技术实现(10 分)：不仅要求综合运用多种 CSS 选择器与现代布局技术（如
Flexbox/Grid），更鼓励在设计中体现独特的创意与视觉表现力，能通过样式提升页面的整
体美感和用户体验。
5. 解决用户痛点问题的能力 (5 分)：图文排版与资源管理需围绕真实用户需求展开，
例如通过响应式设计解决多端适配问题，图片资源合理引入与加载优化，提升页面可用性和
访问效率。
6. 组件化思维与布局合理性 (10 分)：除了将 Header、Sidebar 等布局内容合理封装
为可复用组件外，还需在答辩中清晰阐述组件划分的意图与布局设计的逻辑，体现对模块化
开发和布局架构的理解。
7. 答辩过程中能流畅说明所使用的技术选型及其优势（10 分），并能够快速定位和展
示关键代码段落，展现对项目结构与样式实现的全面掌握。
8.Vue 表单控件与双向绑定 (10 分)：网站中包含表单，至少使用五种不同类型的表单
控件（如 input、textarea、select、checkbox、radio）。能熟练运用 v-model 指令实现表
单数据的双向绑定，并理解其原理。
9.Vue 交互与动态效果 (10 分)：能正确应用 Vue 的指令（如 v-if、v-show、v-for）
和事件处理（v-on 或@）设计合理的动态交互效果。鼓励使用 Vue 的过渡&动画系统
（Transition/TransitionGroup）实现平滑的效果。
10.项目完整度与美观性 (10 分)：页面内容充实，设计美观，层次清楚。网站导航、
链接路由跳转等功能简便易用。图片处理得当，无明显拉伸失真。整个应用运行流畅，用户
体验良好。
11.项目文档与代码规范 (10 分)：项目报告（或 README.md）撰写规范，条理清晰地描
述 Vue 项目的搭建、组件设计思路和功能实现步骤。代码结构清晰，符合 ESLint 等规范，
Vue 单文件组件内代码缩进正确，关键逻辑有注释，能充分体现对 Vue 开发流程的理解。
